# 📊 Оптимизация и партиционирование в PostgreSQL

Этот скрипт демонстрирует основные методы оптимизации запросов в PostgreSQL: генерацию тестовых данных, создание индексов (B-Tree и GIN), партиционирование таблиц и работу с типом данных JSONB.

## 🚀 Быстрый старт

1. Скопируйте SQL-код из файла.
2. Выполните его в вашей базе данных PostgreSQL.
3. Изучите планы выполнения запросов (`EXPLAIN ANALYZE`).

---

## 📝 Описание шагов

### 1. Создание и наполнение таблицы
* Создается таблица `events` со связкой пользователя, типа события и времени.
* Генерируется **100 000** случайных записей за 2025 год.

### 2. Индексация (B-Tree)
* **Поиск без индекса:** последовательное сканирование всей таблицы (`Seq Scan`).
* **Поиск с индексом:** создается индекс `user_id_idx`, ускоряющий поиск конкретного пользователя.
* **Составной индекс:** создается `user_id_comp_idx` для демонстрации работы с диапазонами дат.

### 3. Партиционирование таблицы
* Создается партиционированная таблица `events_part` с разделением по диапазонам дат (`RANGE`).
* Создаются 3 партиции на первые месяцы 2025 года.
* Демонстрируется механизм `Partition Pruning` — БД читает только нужную партицию, игнорируя остальные.

### 4. Работа с JSONB и GIN-индексами
* В таблицу `events` добавляется колонка `metadata` в формате JSONB.
* Создается специализированный **GIN-индекс** для быстрого поиска по ключам и значениям внутри JSON.

## ⚠️ Возможные ошибки при выполнении

При запуске скрипта "как есть" вы столкнетесь со следующими ошибками:

1. **Ошибка отсутствия партиции:** Последняя вставка на дату `'2025-05-01'` упадет, так как для мая нет созданной партиции.
